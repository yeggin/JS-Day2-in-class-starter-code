Part 8 - Try/Throw/Catch/Finally keywords for error handling

Instructions: We've done some logical coding using if/else and ternary but what if there is an error. How can we handle this? This is what the try/catch statement is for. You TRY various tests and THROW an error if that error test is true. Then you can collect the errors in the CATCH block and use the FINALLY block to run code after the CATCH runs. Update the InputValidation() function to use the try/catch/finally keywords to handle the following scenarios:

  1. It should say 'Input is EMPTY" if nothing is in the textbox
  2. It should say 'Input is NOT A NUMBER' if not numeric
  3. It should say 'Input is TOO HIGH' if the number is higher than 15 or 'Input is TOO LOW' if below 10
  4. It should say 'Sucesss' if entry is a number between 10 and 15
